# refine_paper · 文献精炼工作台

## 0. 核心定位

本 Skill 是一个**对话驱动的文献精炼工具**。

它的核心任务是：

> 用户提供一批论文 PDF，本 Skill 读取论文后，通过与用户对话确认精炼方式，然后按确认的方式对文献进行精炼输出。

**精炼方式不是预设的，而是在对话中与用户共同确定。**

本 Skill 不预设"必须蒸馏某位学者"或"必须生成 Skill"。精炼的粒度、维度和输出形式，完全由用户在对话中指定。

---

## 1. 最高原则

### 1.1 用户指定文献优先

用户上传的论文 PDF 是本 Skill 的最高价值来源。

不得在用户未提供论文的情况下凭空精炼。

### 1.2 精炼方式由对话确认

不得跳过"确认精炼方式"这一步。

必须先问用户"你想怎么精炼"，再动手。

### 1.3 证据分层

所有精炼输出必须区分：

1. 论文原文明确写到的；
2. 多篇论文反复出现、可归纳的；
3. 基于论文结构和研究经验的合理推断；
4. 助手提出的外部建议。

不得把 3、4 类内容伪装成论文原文观点。

### 1.4 不编造

不得编造论文没写过的数据、结论或方法。

材料不足时，必须标注"原文未充分说明"或"置信度较低"。

# Phase 0：入口分流

收到用户输入后，先判断任务类型。

## 0.1 任务类型判断

| 用户输入                    | 路径                   | 示例                                     |
| --------------------------- | ---------------------- | ---------------------------------------- |
| 上传论文 PDF + 说明精炼需求 | Phase 0A：直接路径     | "帮我精炼这几篇论文""读这些论文帮我整理" |
| 上传论文 PDF 但没说怎么精炼 | Phase 0B：对话确认路径 | "这些 PDF 帮我看看"                      |
| 要求更新已有精炼结果        | Phase 0C：更新路径     | "我又加了两篇新论文，更新一下"           |

读取论文标题、摘要、关键词，判断研究领域。

**领域检测的目的**：辅助推荐精炼方式，但不强制选择。

| 领域       | 特征                             | 可能相关的精炼方式                     |
| ---------- | -------------------------------- | -------------------------------------- |
| 公司金融   | 企业投资/融资/治理/并购/盈余管理 | 研究范式蒸馏（需同作者）、实证策略对比 |
| 劳动经济学 | 工资/就业/人力资本               | 方法论提取、文献脉络梳理               |
| 发展经济学 | RCT/DID/政策评估                 | 实证策略对比、研究缺口识别             |
| 宏观经济学 | 货币政策/财政政策/经济增长       | 理论框架整合、结论综合                 |
| 其他       | [具体说明]                       | 由用户在对话中确认                     |

# Phase 0A：直接路径（用户已说明精炼需求）

## 0A.1 触发条件

用户上传了论文 PDF，并且明确说了想怎么精炼。

典型输入：

- "帮我精炼这几篇论文的研究方法"
- "读这些论文，整理出共同的理论框架"
- "帮我拆解这篇论文的识别策略"
- "这几篇论文有什么共同的研究缺口"
- "帮我蒸馏这位老师的研究范式"

## 0A.2 必须确认的信息

### 1. PDF 文件路径

如果用户没有明确给出 PDF 文件的路径，必须确认**绝对路径**。

```markdown
请提供待处理 PDF 文件的**绝对路径**。

⚠️ 只接受绝对路径（例如 `C:\Users\...\paper.pdf`），不接受相对路径。
```

### 1. 确认精炼方式

复述用户的精炼需求，确认理解正确。

```markdown
我理解你想对这些论文进行以下精炼：

- 精炼方式：[用户指定的方式]
- 论文数量：[N] 篇
- 领域判断：[领域名称]

请确认是否正确？
```

### 2. 确认输出形式

询问用户希望的输出形式。

| 输出形式             | 适用场景                 |
| -------------------- | ------------------------ |
| 结构化 Markdown 文件 | 需要保存、反复查阅       |
| 直接在对话中输出     | 快速查看，不需要存档     |
| 生成 Skill 文件      | 蒸馏研究范式，可反复调用 |

默认输出形式：结构化 Markdown 文件。

### 3. 确认精炼粒度

**必须询问用户，不得使用默认值。**

| 粒度                   | 说明                                                       | 输出                                  |
| ---------------------- | ---------------------------------------------------------- | ------------------------------------- |
| **综合提炼**     | 跨论文提取共性模式，不逐篇单独输出                         | 一个共性提炼文件（如 01-研究设计.md） |
| **多篇分别拆解** | 每篇论文按同一个模板逐篇拆解，所有结果合并在一个 md 文件中 | 一个合并拆解文件（如 拆解汇总.md）    |

推荐确认语：

```markdown
你想用哪种精炼粒度？

- **综合提炼**：跨论文提取共性模式，输出一个共性文件
- **多篇分别拆解**：每篇论文按同一个模板逐篇拆解，所有结果合并在一个 md 文件中

请选择一种。
```

### 4. 输出精炼模板供用户修改

确认精炼方式和粒度后，**在正式精炼之前**，先根据用户选择的精炼方式，输出一个精炼模板 md 文件供用户修改。

这一步的目的是：

1. 让用户确认精炼维度是否符合需求；
2. 节约后续逐篇精炼时的上下文开销——模板一旦确认，后续只需按模板执行。

推荐确认语：

```markdown
根据你选择的精炼方式，我先输出一个精炼模板供你修改。

模板文件：`精炼模板-[方式].md`

请你检查模板中的维度是否符合你的需求，可以增删改。确认后我按此模板对所有论文进行精炼。
```

模板生成规则：

| 精炼方式                   | 模板内容                                                               |
| -------------------------- | ---------------------------------------------------------------------- |
| 单篇拆解类（多篇分别拆解） | 每篇论文的拆解维度（元信息、研究问题、数据、变量、模型、识别、结果等） |
| 共性提炼类（综合提炼）     | 共性维度（研究领域、问题意识、数据来源、识别策略、写作范式等）         |
| 其他                       | 由用户自定义维度                                                       |

# Phase 0B：对话确认路径（用户未说明精炼方式）

## 0B.1 触发条件

用户上传了论文 PDF，但没有说想怎么精炼。

典型输入：

- "这些 PDF 帮我看看"
- "帮我读一下这几篇论文"
- "这些论文帮我整理一下"

## 0B.2 对话确认流程

**第一步：读取论文基本信息**

先读取文件名、标题、作者、摘要和关键词。

**第二步：报告论文概况**

```markdown
我已读取以下论文：

| # | 标题 | 作者 | 年份 | 领域 |
|---|---|---|---|---|
| 1 | ... | ... | ... | ... |
| 2 | ... | ... | ... | ... |
| ... | ... | ... | ... | ... |

这些论文涉及的领域主要是：[领域名称]
```

**第三步：询问精炼方式**

```markdown
你想对这些论文进行什么类型的精炼？

以下是常见的精炼方式，你可以选择一个或多个，也可以用自己的话描述：

### 常见精炼方式

| 精炼方式 | 说明 | 适用场景 |
|---|---|---|
| **单篇深度拆解** | 逐篇拆解研究问题、数据、变量、模型、识别策略 | 想深入理解每篇论文 |
| **研究范式蒸馏** | 从同一作者的多篇论文中提炼稳定的研究范式 | 同一作者的多篇论文，想学习其研究风格 |
| **方法论提取** | 提取论文使用的研究方法、识别策略、统计技术 | 想学习论文的方法论 |
| **文献脉络梳理** | 梳理多篇论文之间的引用关系、理论演进、学术脉络 | 想理解一个领域的文献发展 |
| **研究缺口识别** | 从现有文献中识别未被解决的问题和未来方向 | 想找选题或研究方向 |
| **理论框架整合** | 整合多篇论文的理论机制，形成统一框架 | 想理解一个理论领域的全貌 |
| **实证策略对比** | 对比多篇论文的识别策略、稳健性检验、内生性处理 | 想比较不同论文的方法优劣 |
| **结论综合** | 综合多篇论文的发现，提炼共识与分歧 | 想了解一个议题的研究现状 |
| **其他** | 用自己的话描述 | — |

你想选哪种方式？（可以选多个，也可以用自己的话描述）
```

**第四步：确认并进入 Phase 0A**

用户选择后，按 Phase 0A 的流程确认精炼方式、输出形式、粒度，并输出精炼模板供用户修改。

---

# Phase 0C：更新已有精炼结果

## 0C.1 触发条件

用户说：

- "我又加了两篇新论文，更新一下"
- "把这几篇新论文补进原来的精炼"
- "重新精炼一下，加了新材料"

## 0C.2 更新规则

1. 读取已有精炼文件；
2. 单独分析新增论文；
3. 对比新旧结果：
   - 强化既有结论；
   - 修正原有判断；
   - 新增发现；
   - 删除不再稳定的过度归纳；
4. 更新精炼时间、论文清单和诚实边界。

---

# Phase 1：论文预处理

## 1.1 PDF 解析

原始 PDF 论文无法直接被分析。必须先将 PDF 批量转换为结构化 Markdown。

> **调用 `pdf-parse` skill 完成此步骤。** 详见 [skills/pdf-parse/SKILL.md](./skills/pdf-parse/SKILL.md)。

该 skill 会：

1. 检查 MinerU Token（`M_TOKEN` 环境变量）是否已配置；
2. 调用 `skills/pdf-parse/pdf_to_md.py` 将 PDF_DIR 中的论文批量转为结构化 Markdown；
3. 输出 `full.md` + `images/` 到 OUT_DIR。

## 1.2 论文范围

**默认使用全部论文。**

只有当用户明确要求筛选时，才进行筛选。

如用户要求筛选，筛选维度：

| 维度               | 优先级  |
| ------------------ | ------- |
| 用户明确关注的论文 | 🔴 最高 |
| 期刊档次           | 🟡 中   |
| 年份               | 🟡 中   |
| 主题覆盖           | 🟢 低   |

## 1.3 完成检查

进入 Phase 2 前必须确认：

- [ ] PDF 解析已完成；
- [ ] `OUT_DIR` 中每个 PDF 对应一个包含 `full.md` 的子文件夹；
- [ ] 失败的 PDF 已记录；
- [ ] 精炼方式已与用户确认；
- [ ] 精炼粒度已与用户确认；
- [ ] 精炼模板已输出，且用户已确认/修改。

---

# Phase 2：按确认方式执行精炼

**本阶段是核心执行阶段。具体做什么，完全取决于 Phase 0 中与用户确认的精炼方式和用户修改后的精炼模板。**

执行规则：

1. 读取用户确认并修改后的精炼模板；
2. 按模板中的维度，对所有论文执行精炼；
3. 输出结构与模板一致。

---

# Phase 3：输出与交付

## 3.1 输出结构

根据确认的精炼方式和用户修改后的模板，生成对应的输出文件。

```
精炼结果/
├── 精炼模板-[方式].md  ← 用户确认的精炼模板（可选保留）
├── 精炼说明.md          ← 精炼方式、论文清单、诚实边界
├── 拆解汇总.md          ← 多篇分别拆解时的合并输出
├── 01-研究设计.md       ← 综合提炼时的共性文件
├── 02-写作范式.md
├── 03-诚实边界.md
└── [其他精炼文件].md    ← 按确认方式生成
```

## 3.2 精炼说明模板

每个精炼结果必须附带一个说明文件：

```markdown
# 精炼说明

## 精炼方式
- 确认的精炼方式：[具体方式]
- 精炼粒度：[综合提炼 / 多篇分别拆解]

## 论文清单
| # | 标题 | 作者 | 年份 | 期刊 |
|---|---|---|---|---|
| 1 | ... | ... | ... | ... |

## 诚实边界
- 覆盖论文：N 篇
- 精炼时间：YYYY-MM-DD
- 未覆盖内容：...

## 后续可扩展
- ...
```

## 3.3 交付说明

最终交付时，向用户说明：

- 精炼方式；
- 精炼了哪些论文；
- 输出了哪些文件；
- 诚实边界；
- 后续如何更新或追加精炼。

---

## 最终目标

本 Skill 的目标是让用户获得一个可反复调用的文献精炼工具：

> 用户指定论文 → 对话确认精炼方式 → 按确认方式精炼输出。

当工作台运行良好时，用户应能做到：

1. 提供一批论文 PDF；
2. 在对话中确认想怎么精炼；
3. 获得结构化的精炼结果；
4. 随着新论文加入，追加精炼；
5. 在不同精炼方式之间灵活切换。
