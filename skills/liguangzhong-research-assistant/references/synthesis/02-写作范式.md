# 写作范式 — 李广众论文共性提炼

## Introduction 结构

| 功能段 | 李广众典型写法 |
|---|---|
| **开场钩子** | 以具体问题/理论争议/经典谜题开场（非政策口号） |
| | 例："How does uncertainty affect investment? This question has long fascinated economists." |
| | 例："Understanding financial market volatility...is a central and yet unresolved problem." |
| **文献综述** | 按理论分支组织（不是按时间罗列），每个分支概括后指出与本文关系 |
| **文献缺口** | 精准指出"前人未做什么"——而非泛泛"研究不足" |
| | 典型句式："However, most studies only find a negative effect...and little research has shown how the interplay of the theoretical elements shapes the relationship." |
| **实证挑战** | 坦率承认识别困难或数据局限，以此凸显研究设计价值 |
| **研究设计** | 先说明数据/制度场景的优势，再介绍方法 |
| **贡献陈述** | 按文献分支组织（对A文献贡献X，对B文献贡献Y），不堆砌"首次" |
| | 典型：对理论文献贡献+对实证文献贡献+对政策启示 |

## 文献组织方式

- **问题驱动**而非方法驱动
- 常见分组逻辑：
  1. 理论基础文献（经典模型/理论预测）
  2. 实证文献（已有发现及其局限）
  3. 本文与该文献的关系（补充/挑战/连接/延展）

## 结果解释风格

- 先报告**经济显著性**（系数含义），再讨论统计显著性
- 跨列解释变化来源，不只说"结果稳健"
- 与理论假说（H1, H2, H3）逐一对应
- 谨慎使用因果词汇：
  - DID/自然实验 → "导致""causally affects"
  - 面板FE → "suggests""is associated with"

## 写作 DNA：可迁移特征

### 标题
- 英文：短标题+副标题，或问题式（"Does...?"）
- 中文：主标题+副标题（以"基于XX的准自然实验"为常见后缀）

### 中文特有写法特点
- 政策背景→理论框架→实证设计→结果→结论
- 理论部分通常会画逻辑框架图
- 稳健性检验详细呈现在正文中（不仅是附录）

## 禁止写法（基于论文反推）

- 不使用空泛的政策套话（"具有重要理论意义和现实意义"）
- 不含没有机制和证据的宏大判断
- 不编造文献引用
- 不把相关性表述为因果（除非识别策略支持）
